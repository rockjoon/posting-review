# @SpringBootApplication과 Auto-Configuration

docs 참고: [https://docs.spring.io/spring-boot/docs/current/reference/htmlsingle/#using-boot-auto-configuration](https://docs.spring.io/spring-boot/docs/current/reference/htmlsingle/#using-boot-auto-configuration)

## @SpringBootApplication?

`@SpringBootApplication`은 `@Configuration`, `@ComponentScan`, `@EnableAutoConfiguration`이 하나로 정의된 Spring Boot의 설정 어노테이션이다.

```java
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
@Documented
@Inherited
@SpringBootConfiguration
@EnableAutoConfiguration
@ComponentScan(excludeFilters = { @Filter(type = FilterType.CUSTOM, classes = TypeExcludeFilter.class),
		@Filter(type = FilterType.CUSTOM, classes = AutoConfigurationExcludeFilter.class) })
public @interface SpringBootApplication {

	@AliasFor(annotation = EnableAutoConfiguration.class)
	Class<?>[] exclude() default {};

	@AliasFor(annotation = EnableAutoConfiguration.class)
	String[] excludeName() default {};

	@AliasFor(annotation = ComponentScan.class, attribute = "basePackages")
	String[] scanBasePackages() default {};

	@AliasFor(annotation = ComponentScan.class, attribute = "basePackageClasses")
	Class<?>[] scanBasePackageClasses() default {};

	@AliasFor(annotation = ComponentScan.class, attribute = "nameGenerator")
	Class<? extends BeanNameGenerator> nameGenerator() default BeanNameGenerator.class;

	@AliasFor(annotation = Configuration.class)
	boolean proxyBeanMethods() default true;
}
```

위 `@SpringBootApplication`을 사용하면 `@ComponentScan` → `@EnableAutoConfiguration` 순으로 빈 등록한다. 즉, 애플리케이션 내에서 정의한 빈 먼저 등록한 후 Spring Boot의 Auto-Configuration으로 각종 빈 들을 등록한다.

### @ComponentScan

먼저 개발자가 정의한 컴포넌트들을 빈으로 등록하기 위해 `@SpringBootApplication`이 있는 클래스를 기준으로 해당 패키지와 하위 패키지의 컴포넌트를 빈으로 등록한다.

### @EnableAutoConfiguation

Spring Boot의 기능인 자동 설정을 담당하는 어노테이션이다. 위 어노테이션이 있으면 `org.springframework.boot:spring-boot-autoconfigure` 패키지의 `META-INF/spring.factories`파일에 있는 클래스를 조건에 따라 빈으로 등록한다.

## Auto-Configuration

# Auto-Configuration

Spring Boot에서는 추가한 의존성에 따라 자동으로 실행할 수 있는 환경을 구축해준다.

이를 사용하기 위해서는 `@SpringBootApplication`이나 `@Configuration`에서 `@EnableAutoConfiguration`이 설정되어 있어야한다.

만약 Spring MVC를 사용한다고 가정하면 Spring Boot에서는 `WebMvcAutoConfiguration`을 통해 Spring MVC 환경을 구축한다.

```groovy
@Configuration(proxyBeanMethods = false)
@ConditionalOnWebApplication(type = Type.SERVLET)
@ConditionalOnClass({ Servlet.class, DispatcherServlet.class, WebMvcConfigurer.class })
@ConditionalOnMissingBean(WebMvcConfigurationSupport.class)
@AutoConfigureOrder(Ordered.HIGHEST_PRECEDENCE + 10)
@AutoConfigureAfter({ DispatcherServletAutoConfiguration.class, TaskExecutionAutoConfiguration.class,
		ValidationAutoConfiguration.class })
public class WebMvcAutoConfiguration {
	// ...
}
```

만약 `org.springframework.boot:spring-boot-starter-web`의존성에 추가한 경우 DisplatcherServlet과 WebMvcConfigurer가 classpath에 포함되기 때문에 위 `WebMvcAutoConfiguration`이 설정된다.

위 `WebMvcAutoConfiguration` 뿐만 아니라 `AopAutoConfiguration`, `DataSourceAutoConfiguration` 등 다양한 AutoConcfiguration을 지원한다. 더 많은 AutoConfiguaration은 다음 [docs](https://docs.spring.io/spring-boot/docs/current/reference/htmlsingle/#auto-configuration-classes)를 참고.

## Replacing Auto-Configuration

Spring Boot에서는 Auto-Configuration으로 설정하는 특정 부분에 대해 개발자가 스스로 설정할 수 있도록 지원한다.

```groovy

public class WebMvcAutoConfiguration {
	// ...
	
	@Bean
	@ConditionalOnMissingBean
	public InternalResourceViewResolver defaultViewResolver() {
		InternalResourceViewResolver resolver = new InternalResourceViewResolver();
		resolver.setPrefix(this.mvcProperties.getView().getPrefix());
		resolver.setSuffix(this.mvcProperties.getView().getSuffix());
		return resolver;
	}

	// ...
}
```

위와 같이 `WebMvcAutoConfiguration`에는 InternalResourceViewResolver를 빈으로 기본적으로 등록한다. 이때 만약 애플리케이션에서 이미 `InternalResourceViewResolver`를 빈으로 등록을 했다면 `@ConditionalOnMissingBean`에 의해 위 코드는 사용되지 않는다.

이렇게 Spring Boot에서는 `@Conditional` 기능을 통해서 Auto-Configuration 적용 등을 조절한다.

## Disabling Auto-Configuration

Spring Boot에서는 자동으로 등록되는 `*AutoConfiguration`를 제외할 수 있도록 지원한다.

```java
import org.springframework.boot.autoconfigure.*;
import org.springframework.boot.autoconfigure.jdbc.*;

@SpringBootApplication(exclude={DataSourceAutoConfiguration.class})
public class MyApplication {
}
```

위와 같이 `@SpringBootApplication`의 exclude를 설정하여 특정 Auto-Configuration을 제외할 수 있다. 위 예시에서는 `DataSourceAutoConfiguration`를 제외한다.

```java
public @interface SpringBootApplication {

	/**
	 * Exclude specific auto-configuration classes such that they will never be applied.
	 * @return the classes to exclude
	 */
	@AliasFor(annotation = EnableAutoConfiguration.class)
	Class<?>[] exclude() default {};

	/**
	 * Exclude specific auto-configuration class names such that they will never be
	 * applied.
	 * @return the class names to exclude
	 * @since 1.3.0
	 */
	@AliasFor(annotation = EnableAutoConfiguration.class)
	String[] excludeName() default {};
	
	// ...

}
```

위 `SpringBootApplication`에서 볼 수 있듯이 exculde와 excludeName은 `EnableAutoConfiguration`의 exclude와 excludeName을 따라간다.

```java
public @interface EnableAutoConfiguration {

	String ENABLED_OVERRIDE_PROPERTY = "spring.boot.enableautoconfiguration";

	/**
	 * Exclude specific auto-configuration classes such that they will never be applied.
	 * @return the classes to exclude
	 */
	Class<?>[] exclude() default {};

	/**
	 * Exclude specific auto-configuration class names such that they will never be
	 * applied.
	 * @return the class names to exclude
	 * @since 1.3.0
	 */
	String[] excludeName() default {};

}
```

때문에 만약 `@SpringBootApplication`을 사용하지 않고 `@EnableAutoConfiguration`을 사용한다면 `@EnableAutoConfiguration`의 exclude나 excludeName을 사용해도 무방하다.

이외에도 `spring.autoconfigure.exclude` 프로퍼티를 통해서도 제외가 가능하다.

```yaml
spring:
	autoconfigure:
		exclude: org.springframework.boot.autoconfigure.jdbc.DataSourceAutoConfiguration
```
